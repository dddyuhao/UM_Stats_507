## Final project: Temporal Cross-Modal Disclosure Alpha from SEC Filings

An end-to-end Python pipeline that converts SEC 10-Q and 10-K filings into a
cross-sectional equity return-ranking signal, and evaluates it under a realistic,
cost-aware weekly portfolio backtest. 

### Headline result

On the 2024 out-of-sample test window (918 filing events, 10-Q and 10-K combined):

| Signal | 20-day pooled IC | 20-day IC on 10-Q only | Portfolio Sharpe (buffered quintile) |
|---|---|---|---|
| B1 – FinBERT sentiment only | −0.024 | −0.047 | +0.02 |
| B2 – Ridge on FinBERT + 8 market features | +0.004 | +0.020 | −0.11 |
| **TCMDA (proposed)** | **+0.033** | **+0.087** (p=0.044) | **+0.18** |
| Equal-weight passive benchmark | — | — | +0.64 |

On the 10-Q subset, TCMDA's monthly cross-sectional IC series has
`mean = +0.0370`, Newey-West t-stat = +5.46, p = 0.003 — the only signal–model–horizon
combination among the three to achieve significant positive IC. The main
(10-Q, buffered quintile, cost-aware) portfolio outperforms both active baselines
while remaining below the passive equal-weight benchmark in 2024, a strong bull market.

An ablation confirms that ~2/3 of TCMDA's realized alpha comes from the
entry/exit turnover buffer: removing it drops Sharpe from +0.18 to −0.49.

### Pipeline at a glance

| Stage | Notebook | What it does |
|---|---|---|
| 1 | `01_data_collection.ipynb` | Downloads ~17k 10-Q/10-K filings for 463 pre-2015 S&P 500 firms; cleans HTML; pulls daily prices via `yfinance`; constructs event-aligned dataset with 5d/20d forward excess returns (market-adjusted) and 8 lagged market features. |
| 2 | `02_finbert_baseline.ipynb` | Scores each filing with `ProsusAI/finbert` (chunked, averaged). Baseline 1 uses net sentiment directly. Baseline 2 fits a Ridge regressor on sentiment + market features. |
| 3 | `03_proposed_model.ipynb` | Extracts FinBERT CLS embeddings → PCA to 64d → trains the TCMDA neural model with a hybrid MSE + negative-Pearson loss on cross-sectional rank targets. Time-ordered train/val/test split (2015–2022 / 2023 / 2024 H1). |
| 4 | `04_portfolio_construction.ipynb` | Weekly backtest with transaction costs. Main result: cross-sectional z-score per event-month → buffered quintile (enter top 15 / exit past rank 30). Reports main result + four diagnostic variants (no-buffer, sector-neutral, market-neutral L/S, sign-flip). |

### TCMDA architecture (Notebook 03)

For each filing `i` of firm `j`, given current embedding `e_i` and previous same-form
embedding `e_{i-1}`:

```
z_i = [ e_i,                                     # current filing (64-d PCA-FinBERT)
        e_i - e_{i-1},                           # temporal change vector (64-d)
        cos(e_i, e_{i-1}),                       # similarity to prior filing (1-d)
        days_since_prior / 365,                  # timing (1-d)
        market_features ]                        # 8 lagged market state features
```

Three projection branches (text, text-change, market) feed into a fusion MLP.
Parameters: ~14.5k. Training target is cross-sectional percentile rank of
forward excess return within event-month (scaled to [−0.5, 0.5]).

Loss: `L = MSE(ŷ, y_rank) + λ · (−Pearson(ŷ, y_rank))`

### Reproducibility notes

The `data/` directory is intentionally **not** committed to this repository
(it is 1.6 GB, and one of its parquet files is 425 MB — GitHub's hard limit
for a single file is 100 MB). Instead, the data is fully **defined by code**:

- `01_data_collection.ipynb` downloads 10-Q and 10-K filings directly from the
  [SEC EDGAR API](https://www.sec.gov/search-filings/edgar-application-programming-interfaces)
  and daily prices via [`yfinance`](https://ranaroussi.github.io/yfinance/).
  Both are public sources; no API keys required.
- `02_finbert_baseline.ipynb` regenerates the FinBERT sentiment scores
  (`finbert_scores.parquet`) on first run.
- `03_proposed_model.ipynb` regenerates the 768-dim FinBERT CLS embeddings
  (`filing_embeddings.npy`) and the 64-dim PCA projection on first run.

On a fresh clone, running notebooks 01 → 04 in order reproduces every
artifact that `data/` would have contained. Each notebook caches its own
outputs, so subsequent runs are fast. End-to-end first run time:

- Notebook 01: ~30 min (most of it is the SEC rate-limit of ≤10 req/sec)
- Notebook 02: ~10 min on a modern NVIDIA GPU
- Notebook 03: ~10 min on GPU
- Notebook 04: ~2 min (pure pandas/numpy)

If a run is interrupted, the caches persist — re-running picks up where it left off.

### Known limitations (also noted in the report)

1. **Filing coverage cutoff.** The SEC filing collection ends on 2024-06-30, while
   daily market data extends through 2024-12-30. Because signals expire after
   45 trading days, the strategy naturally winds down to cash in late 2024 —
   this is a sample-design artifact, not a signal failure.
2. **Short test window.** The 2024 test window contains only 918 events in 6 months,
   which limits statistical power (especially for Newey-West time-series tests,
   hence the auto-adjusted lag logic in notebook 04).
3. **Passive benchmark asymmetry.** The equal-weight benchmark always holds all
   463 names; the TCMDA portfolio averages 12 names. In a bull market this
   structural difference dominates any alpha.
4. **Chunked text encoding.** FinBERT embeddings come from 8 chunks × 400 tokens;
   a 10-K averages 30k+ tokens, so >70% of each document is unseen. A long-document
   encoder (Longformer, FinBERT-Long) or section-level parsing would likely improve IC.
5. **Survivorship bias.** Filtering to pre-2015 S&P 500 members excludes firms
   that joined later or were delisted. A commercial constituent-membership
   database would fix this; free alternatives do not exist.
